# Mayday Life 后台设计规范

本规范只适用于 `/admin/**`、`app/layouts/admin.vue` 和 `Admin*` 组件。前台页面不受本文档的管理端密度和布局约束影响。

`MUST` 是合并前必须满足的规则。`SHOULD` 只能在明确的业务限制下偏离，并需在代码中保留可理解的原因。视觉迁移必须保留现有 API、请求载荷、保存、权限和错误处理行为。

## 1. 视觉与布局

- **MUST** 采用安静、克制、信息密集的 CMS 风格。管理集合默认使用表格，不改成卡片网格。
- **MUST** 禁止嵌套卡片、装饰性渐变、玻璃效果、悬浮抬升和营销式大标题。仪表盘指标卡是例外。
- **MUST** 保留侧边栏 `240px`、折叠宽度 `80px`、顶栏 `52px` 和页面最大宽度 `1600px`。
- 页面横向留白为移动端 `16px`、中屏 `20px`、大屏 `24px`；常规区块间距 `16px`，大区块 `24px`。
- 页面本身不得产生横向滚动；窄屏表格只允许表格容器自身滚动。

## 2. 组件与样式

- **MUST** 先检查 `app/components/ui` 并优先使用现有 `Ui*` 组件。后台页面不得直接导入 `reka-ui`。
- 隐藏文件输入、TinyMCE 内部控件以及经明确封装的底层 `Ui*` 组件可直接使用原生或 Reka primitive。
- 为 `UiInput`、`UiSelectTrigger`、`UiButton`、`UiTextarea` 附加的 class 只能控制宽度、定位、外边距、flex/grid 布局和响应式显隐。
- **MUST NOT** 在页面覆盖基础控件的圆角、边框、背景、颜色、阴影、焦点、内边距或高度。重复需求必须下沉为组件 `variant` 或 `size`。
- 图标统一使用 Lucide。图标按钮必须同时具有 `aria-label` 和 Tooltip。

## 3. 操作和菜单

- 每个页面最多一个主操作。次要操作使用 `outline` 或 `ghost`，危险操作使用 `destructive`。
- 1–2 个动作直接显示；3 个相关动作使用 `UiButtonGroup`。
- 4 个及以上动作只保留前两个高频动作，其余放入“更多” `UiDropdownMenu`。移动端只保留主操作和菜单。
- 表格行最多显示两个高频图标操作，其余进入下拉菜单。危险项置于菜单末尾并用分隔线隔开。
- **MUST** 使用 `UiAlertDialog` 确认删除和其他破坏性操作，禁止 `window.confirm`。
- 异步按钮必须在请求中禁用并显示进度，不得因文案变化导致宽度跳动。

## 4. 表格与分页

- **MUST** 使用 `UiTable`。管理列表默认每页 `20` 条；仪表盘最近记录可保持 `5` 条。
- 首列用于识别对象，状态使用文字加 Badge，数字右对齐并使用 `tabular-nums`，操作列固定在最右。
- 长文本截断时必须提供完整内容的 title 或 Tooltip。优先在窄屏隐藏次要列，再启用表格容器滚动。
- 筛选条件变更后重置到第 1 页；新增、删除或数据刷新后必须将页码钳制在有效范围。
- loading、empty 和 error 状态必须在表格内保持稳定高度；error 状态需要提供重试操作。

## 5. 表单与弹窗

- 设置和编辑表单统一使用标签在上、控件在下的单列布局，字段间距 `16px`，标签与控件间距 `8px`。
- 真实表单控件区域默认 `max-w-xl`。筛选栏和表格操作组不属于真实表单，可保持横向紧凑布局。
- 字段必须有可见 label；必填字段有明确标记；帮助文本和错误就近显示；提交失败后焦点移动到第一个无效字段。
- 确认弹窗使用 `size="sm"`，普通 CRUD 使用 `size="md"`，长表单使用 `size="lg"`，复杂图册编辑最多使用 `size="xl"`。
- 页面不得为 Dialog 直接写 `max-w-*`。Dialog 最大高度为 `calc(100dvh - 32px)`，内容区滚动，标题和底部操作保持可见。
- Dialog 必须支持 Escape、关闭按钮、初始焦点和关闭后焦点归还。有未保存修改时需确认后才可关闭。

## 6. 主题、状态与动效

- 保留现有蓝灰配色、亮暗主题和字体。使用语义 token，页面不得硬编码主题色或状态色。
- 必须定义 surface hover、success、warning、danger、info 及对应 soft token，并同时提供亮暗主题值。
- 路由跳转使用现有顶部进度条。结构化内容等待超过 `300ms` 使用 Skeleton，按钮等局部操作使用 spinner。
- Toast 只用于瞬时成功、警告或错误反馈，不能代替页面内的错误提示和重试操作。
- 微交互使用 `150–200ms`，弹窗进入不超过 `200ms`、退出更短，只动画 opacity 和 transform。
- **MUST** 在 `prefers-reduced-motion` 下关闭非必要动画。

## 7. 响应式与无障碍

- 固定验收宽度为 `375px`、`768px`、`1024px` 和 `1440px`。`1024px` 以下侧边栏使用抽屉。
- 普通文字对比度至少 `4.5:1`，大文字和 UI 图形至少 `3:1`。不得只使用颜色表示状态。
- 桌面端紧凑按钮可为 `32px` 或 `36px`；在粗指针设备上点击区域至少 `44×44px`。
- 所有功能必须可用键盘操作，焦点环清晰，Dialog、Dropdown、Toast 和表格状态具有正确语义。
- 后台 shell 必须提供“跳到主内容”链接。路由切换后应将焦点移入主内容。

## 8. 验收清单

- 后台页面无直接 `reka-ui` import、无 `window.confirm`、无可见原生控件绕过、无基础控件视觉覆盖。
- `UiButtonGroup`、`UiDropdownMenu`、`UiDialog`、`UiAlertDialog`、`UiTooltip`、`UiSkeleton` 和 `UiAlert` 存在并在后台统一使用。
- 验证仪表盘、文章列表/编辑器、图片、图册、分类、评论、站点设置、功能设置、登录和初始化流程。
- 验证键盘、Escape、异步禁用、错误重试、空状态、分页钳制、亮暗主题和 reduced motion。
- 提交前运行目标文件 ESLint、`pnpm exec nuxi prepare`、后台路径类型检查、`git diff --check` 和 `pnpm run build`。
